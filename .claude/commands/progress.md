# /progress - 查看详细项目进度

1. 扫描项目目录
   - 后端：`src/` 或 `ruoyi-business/` 等
   - 前端：`src/views/`
   - 移动端：`pages/`
2. 统计模块完整度
   - 检查 Entity/Service/DAO/Controller 或页面/路由/接口
3. 计算完成度
   - 100%：组件齐全
   - 85%：缺 Controller/Page
   - 70%：缺 Service/逻辑
   - 50%：仅数据结构
   - 20%：只有目录
4. 读取 `docs/项目状态.md`、`docs/待办清单.md`
5. 汇总 TODO/FIXME 与代码质量

## 输出格式
```
# 📊 项目进度报告

## 🎯 项目概况
- 项目名称：<>
- 技术栈：<>
- 当前阶段：<>
- 整体进度：<XX%>

## 📈 功能模块进度
### 后端模块
#### <模块> (<完成度>)
1. <功能>
   - Entity: <状态>
   - Service: <状态>
   - DAO: <状态>
   - Controller: <状态>

### 前端页面（已实现 <数量> 个）
1. <页面> - <路由>
   - 位置: <path>

### 移动端页面（已实现 <数量> 个）
...

## 📝 待完成事项
- [ ] 任务 1
- [ ] 任务 2

## 🔍 代码质量分析
- 代码规范性：<>
- 注释完整度：<>
- TODO/FIXME：<数量>
```
