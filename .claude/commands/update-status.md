# /update-status - 自动更新项目状态

1. 读取 `docs/项目状态.md` 与 `docs/待办清单.md`
2. 查看最近 10 条 `git log`，确认新增内容
3. 扫描后端/前端/移动端目录，统计完成度
4. 统计 TODO/FIXME 变化
5. 生成更新报告：
   - 新增已完成任务
   - 更新进行中任务
   - 新增或关闭待办
   - 整体进度变化
6. 写回文档，保留人工备注

## 输出格式
```
# 📝 项目状态更新报告

## 本次更新内容
- ✅ 新增已完成任务：<数量，列表>
- 🚧 更新进行中任务：<数量>
- 📋 新增待办任务：<数量>

## 主要变更
1. <任务>：<状态变化>（Git: `<commit>`）
2. ...

## 📊 进度变化
- 整体进度：<旧值> → <新值>
- 模块 A：<旧值> → <新值>

---

✅ 文档已更新：`docs/项目状态.md`
```

## 注意事项
- 如无变更必须说明“无更新”原因
- 不得覆盖用户在文档添加的自定义段落，可在文末附加
