# /next - 获取下一步建议

1. 查看最近 5 条 Git 提交，判断阶段
2. 扫描 TODO/FIXME (`rg --no-heading --line-number "TODO|FIXME"`)
3. 阅读 `docs/项目状态.md` 与 `docs/待办清单.md`
4. 识别阻塞关系（如 Service 完成但 Controller 未完成）
5. 根据规则排序：
   - 🔴 高优先级：FIXME、阻塞、生产缺陷
   - 🟡 中优先级：已开始未完成、TODO、计划内功能
   - 🟢 低优先级：重构、性能优化、文档
6. 输出可执行步骤

## 输出格式
```
# 🎯 下一步建议

## 📌 当前状态
- 最近在做：<git 总结>
- 开发阶段：<初期/中期/收尾>
- 代码状态：<正常/有未提交>

## 🚀 建议优先级
### 🔴 高优先级
1. **任务**
   - 原因：<>
   - 影响：<>
   - 预计耗时：<>
   - 位置：<path>
   - 建议：<步骤>

### 🟡 中优先级
...

### 🟢 低优先级
...

## 💡 具体行动建议
- 第一步：<描述 + 依赖>
- 第二步：<描述>

## 📝 注意事项
- ⚠️ 事项 1
- ⚠️ 事项 2
```
